Part 1 Data Preparation¶

I used Safe Software FME to perform the data cleansing. I imported the CSV file, performed a "Tester" to ensure the Lat/Longs were correct. I set the values of >= 39 for Lat and >=80 for Long. The next transformer I used was a DuplicateFilter to filter out duplicate lat and long key attributes. Next I ran an AttributeFilter to split local_dtm into a day and time. I ran it through another AttributeFilter to split out the Day, Month and Year. Next the process went through an AttributeManager where I set the Date to "@Value(Date{1})-@Value(Date{0})-20@Value(Date{2})" i.e, MAY-19-2017. Next the process trims the ":" in front of the Time field. I run a TestFilter to extract the unique "trip_nb" values (16). Next I perform a DateTimeConvertor to put the processed DateTime field into a format that can be used for Calculating Motion Tracks in ArcGIS Pro ("2017-05-19 07:33:59"). Next I run another DuplicatFilter to remove duplicate DateTimes so the geoprocessing tool in ArcGIS Pro will not error.

In ArcGIS Pro, I brought in the output CSV from FME. I ran a process to Display X/Ys from the dataset into the correct coordinate system in the Map. I used the Calculate Motion Statistic toolset through GeoAnalytics Desktop geoprocessing tools. This allows the user to calculate motion statistics such as Acceleration, Speed, Distance, Idling, Bearing, and many others. The tool allows the user to set certain thresholds on idle distance, idle time, and set a track history window consisting of a set-amount of previous points to use in the analysis.
